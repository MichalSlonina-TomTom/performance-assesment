# Performance Expectations by Level

This directory contains detailed performance expectations for different software engineering levels at TomTom. These documents provide comprehensive role summaries, responsibilities, and expectations for each level.

## Available Levels

- [Software Engineering Level 2](./software_engineering_level_2.md) - Software Engineer I
- [Software Engineering Level 3](./software_engineering_level_3.md) - Software Engineer II
- [Software Engineering Level 4](./software_engineering_level_4.md) - Software Engineer IV

## How to Use These Documents

These performance expectations documents complement the [three pillars](../README.md) of TomTom's performance framework:
- [Leadership](../pillars/leadership.md)
- [Capabilities and Development](../pillars/capabilities_and_development.md)
- [Impact](../pillars/impact.md)

Each level document includes:
- **Job Family Summary** - Overview of the software engineering discipline
- **Role Summary** - Key attributes including:
  - Knowledge requirements and experience
  - Problem-solving expectations
  - Training and mentoring responsibilities
  - Accountability and independence levels
  - Stakeholder relationships
- **Job Description** - Detailed narrative of the role's expectations and scope

Use these documents alongside your self-assessment to understand the expectations for your current level and aspirations for growth.
